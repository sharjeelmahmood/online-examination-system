# Software Requirements Specification (SRS)

# 1. Introduction

The Online Examination System is designed to conduct online exams securely and efficiently.

# 2. Objectives

- Conduct online examinations
- Prevent multiple submissions
- Generate automatic results
- Maintain exam security

# 3. Functional Requirements

1. Student Login
2. Start Exam
3. Submit Exam
4. Generate Result
5. Admin Exam Management

# 4. Non Functional Requirements

- Security
- Reliability
- Fast response time
- Data integrity

# 5. Actors

| Actor | Description |
|---|---|
| Student | Attempts exam |
| Admin | Manages exams |
| Examiner | Creates questions |

# 6. System Workflow

1. Student logs in
2. Student starts exam
3. Student answers questions
4. Student submits exam
5. System generates result
